# 新增学习示例说明 (2024-03)

## 本次更新内容

为项目新增了 **4个重要的Java学习示例**，这些内容对理解Spring框架至关重要：

### 1. DateTimeDemo.java - Java 8日期时间API

**位置**: `src/advanced/DateTimeDemo.java`

**学习要点**:
- ✅ LocalDate/LocalTime/LocalDateTime的使用
- ✅ 日期时间的格式化和解析
- ✅ 日期计算和Period/Duration
- ✅ 时区处理（ZonedDateTime）

**为什么重要**:
- Spring应用中处理订单日期、用户生日、创建时间等
- REST API返回日期格式需要正确序列化
- 国际化应用必须掌握时区处理
- @DateTimeFormat和@JsonFormat注解的基础

**运行示例**:
```bash
cd src && java advanced.DateTimeDemo
```

---

### 2. OptionalDemo.java - Optional类

**位置**: `src/advanced/OptionalDemo.java`

**学习要点**:
- ✅ Optional的创建（of、ofNullable、empty）
- ✅ 获取值的安全方法（orElse、orElseGet）
- ✅ Optional的转换（map、flatMap、filter）
- ✅ 实战：用户服务查询示例

**为什么重要**:
- **Spring Data JPA核心特性**：`findById()`返回`Optional<T>`
- 避免空指针异常，让代码更安全
- REST API中优雅处理不存在的资源
- Spring配置中处理可选配置项

**运行示例**:
```bash
cd src && java advanced.OptionalDemo
```

---

### 3. EnumDemo.java - 枚举类型

**位置**: `src/advanced/EnumDemo.java`

**学习要点**:
- ✅ 基本枚举和枚举方法
- ✅ 带属性的枚举（HTTP状态码示例）
- ✅ 带方法的枚举（计算器操作示例）
- ✅ 实战：订单状态管理系统

**为什么重要**:
- 定义业务状态（订单状态、支付状态、用户状态）
- 定义用户角色和权限
- JPA中使用`@Enumerated`映射到数据库
- REST API中作为请求参数和响应值
- 比常量类更类型安全

**运行示例**:
```bash
cd src && java advanced.EnumDemo
```

---

### 4. RegexDemo.java - 正则表达式

**位置**: `src/advanced/RegexDemo.java`

**学习要点**:
- ✅ 正则表达式基础语法
- ✅ Pattern和Matcher的使用
- ✅ 常用验证模式（邮箱、手机号、身份证、URL、IP）
- ✅ 实战：用户注册表单验证

**为什么重要**:
- **Spring Validation核心**：`@Pattern`注解验证表单输入
- `@Email`和`@URL`等注解的底层实现
- Spring MVC的RequestMapping路径匹配
- Spring Security的URL权限匹配
- 数据清洗和脱敏

**运行示例**:
```bash
cd src && java advanced.RegexDemo
```

## 学习建议

### 学习顺序

1. **先学Optional** → 掌握安全处理null的方法
2. **再学Enum** → 理解类型安全的常量定义
3. **然后学DateTime** → 掌握日期时间处理
4. **最后学Regex** → 掌握数据验证

### 与Spring的关系

这4个示例都是**Spring开发的必备知识**：

| 示例 | Spring应用场景 | 关联注解/特性 |
|------|--------------|-------------|
| Optional | Repository查询、配置管理 | `Optional<T>` 返回类型 |
| Enum | 状态管理、角色权限 | `@Enumerated` |
| DateTime | 数据时间戳、API返回 | `@DateTimeFormat`, `@JsonFormat` |
| Regex | 表单验证、路径匹配 | `@Pattern`, `@Email` |

### 实战练习建议

每个示例都包含完整的**实战案例**，建议：

1. **先运行示例**：理解输出结果
2. **阅读注释**：每段代码都有详细的中文说明
3. **修改代码**：尝试自己写验证逻辑
4. **结合Spring**：思考如何在Spring项目中使用
